## 二、各功能模块详细设计

### 2.1 工作台 (Dashboard)

#### 核心功能点
1. **数据概览卡片**
   - 学生总数统计
   - 待审进度数量（带红点提醒）
   - 未读消息数量
   - 未读通知数量

2. **快速访问区域**
   - 学生列表（显示前4名）
   - 待审进度列表（显示前3条）
   - 即将到来的预约（显示前3条）
   - 近期任务（显示前3条）

3. **快捷操作**
   - 一键跳转到各功能模块
   - 快速发起消息
   - 快速创建任务

#### 数据流向
```
前端组件 (GlassMentorDashboard)
    ↓
API调用 (并发请求)
    ├─ myApi.getMyStudents()           → GET /my/students
    ├─ myApi.getMyPendingProgress()    → GET /my/pending-progress
    ├─ taskApi.getTasks()              → GET /tasks/
    ├─ appointmentApi.getAppointments()→ GET /appointments/
    ├─ notificationApi.getNotifications()→ GET /notifications/
    └─ messageApi.getMessages()        → GET /messages/
    ↓
后端处理
    ├─ 验证JWT Token
    ├─ 查询当前导师的学生
    ├─ 查询待审进度
    ├─ 查询任务/预约/消息/通知
    └─ 返回聚合数据
    ↓
前端渲染
    ├─ 统计卡片显示数量
    ├─ 列表显示详细信息
    └─ 动画效果展示
```

#### 前后端交互
| 操作 | 前端 | 后端API | 数据格式 |
|------|------|---------|----------|
| 加载数据 | loadDashboardData() | GET /my/students<br>GET /my/pending-progress<br>GET /tasks/<br>GET /appointments/<br>GET /notifications/<br>GET /messages/ | { items: [], total: number } |
| 跳转详情 | navigate('/my-students/:id') | - | - |

#### 状态管理
- **本地状态**: students, pendingProgress, tasks, appointments, notifications, messages, isLoading
- **全局状态**: user (from AuthContext)

### 2.2 学生管理 (My Students)

#### 核心功能点
1. **学生列表视图**
   - 学生基本信息卡片（姓名、学号、年级、专业）
   - 研究课题显示
   - 搜索过滤功能

2. **学生详情视图**
   - 基本信息展示（性别、入学日期、学生类型）
   - 研究信息展示（课题、专业、年级）
   - 进度历史时间线
   - 快捷操作（发送消息）

3. **进度历史**
   - 进度列表（标题、内容、完成度、状态）
   - 进度详情展开
   - 问题与计划展示

#### 数据流向
```
列表视图
    ↓
GET /my/students
    ↓
返回学生数组 [{id, name, student_no, grade, major, research_topic, ...}]
    ↓
前端渲染卡片列表

详情视图
    ↓
GET /my/students/:id
    ↓
返回 { student: {...}, progress_history: [...] }
    ↓
前端渲染详情页 + 进度时间线
```

#### 前后端交互
| 操作 | 前端方法 | 后端API | 请求参数 | 响应数据 |
|------|----------|---------|----------|----------|
| 加载学生列表 | loadStudents() | GET /my/students | - | Student[] |
| 加载学生详情 | loadStudentDetail(id) | GET /my/students/:id | - | { student, progress_history } |
| 搜索学生 | filterStudents() | - | - | (前端过滤) |

---

### 2.3 进度审阅 (Pending Progress)

#### 核心功能点
1. **待审列表视图**
   - 进度卡片（标题、内容摘要、完成度、提交时间）
   - 学生信息显示
   - 快速审阅按钮

2. **审阅详情视图**
   - 学生信息展示
   - 进度详细内容
   - 遇到的问题
   - 下一步计划
   - 完成度进度条

3. **反馈表单**
   - 星级评分（1-5星）
   - 审核结果（通过/需修改）
   - 反馈内容文本框
   - 提交反馈

4. **AI辅助审阅（扩展功能）**
   - AI分析进度质量
   - 智能建议反馈内容
   - 关键问题识别

#### 数据流向
```
列表视图
    ↓
GET /my/pending-progress
    ↓
返回待审进度数组 [{id, student_id, title, content, completion, ...}]
    ↓
前端渲染待审列表

审阅详情
    ↓
GET /progress/:id
    ↓
返回 { progress: {...}, feedback: {...} }
    ↓
前端渲染详情 + 反馈表单

提交反馈
    ↓
POST /progress/:id/feedback
    ↓
{ content, rating, is_approved }
    ↓
更新进度状态为 'reviewed'
    ↓
创建/更新反馈记录
    ↓
返回反馈详情
```

#### 前后端交互
| 操作 | 前端方法 | 后端API | 请求体 | 响应数据 |
|------|----------|---------|--------|----------|
| 加载待审列表 | loadPendingProgress() | GET /my/pending-progress | - | ProgressReport[] |
| 加载进度详情 | loadProgressDetail(id) | GET /progress/:id | - | { progress, feedback } |
| 提交反馈 | handleSubmitFeedback() | POST /progress/:id/feedback | { content, rating, is_approved } | MentorFeedback |

### 2.4 任务管理 (Task Management)

#### 核心功能点
1. **任务列表视图**
   - 任务卡片（标题、描述、优先级、截止日期）
   - 优先级标签（高/中/低）
   - 分配学生列表

2. **任务创建**
   - 任务标题和描述
   - 优先级选择
   - 截止日期设置
   - 分配学生（多选）

3. **任务详情**
   - 任务基本信息
   - 分配列表（学生、状态、提交内容、反馈）
   - 学生提交情况
   - 导师反馈功能

4. **任务跟踪**
   - 学生完成状态
   - 提交内容查看
   - 反馈提交

#### 数据流向
```
任务列表
    ↓
GET /tasks/
    ↓
根据角色返回任务列表
    ├─ 导师: 返回自己创建的任务
    └─ 学生: 返回分配给自己的任务
    ↓
前端渲染任务列表

创建任务
    ↓
POST /tasks/
    ↓
{ title, description, priority, due_date, student_ids }
    ↓
创建任务记录
    ↓
创建任务分配记录（TaskAssignment）
    ↓
返回任务详情

任务详情
    ↓
GET /tasks/:id
    ↓
返回任务 + 分配列表
    ↓
前端渲染详情页

更新分配状态
    ↓
PUT /tasks/:id/assignments/:assignment_id
    ↓
学生: { status, submission_content }
导师: { feedback, status }
    ↓
更新分配记录
    ↓
返回更新后的分配信息
```

#### 前后端交互
| 操作 | 前端方法 | 后端API | 请求体 | 响应数据 |
|------|----------|---------|--------|----------|
| 加载任务列表 | loadTasks() | GET /tasks/ | - | Task[] |
| 创建任务 | createTask() | POST /tasks/ | { title, description, priority, due_date, student_ids } | Task |
| 加载任务详情 | loadTaskDetail(id) | GET /tasks/:id | - | Task + assignments |
| 更新分配 | updateAssignment() | PUT /tasks/:id/assignments/:assignment_id | { status, submission_content, feedback } | TaskAssignment |

### 2.6 消息中心 (Message Center)

#### 核心功能点
1. **消息列表**
   - 对话列表（最近联系人）
   - 未读消息数量
   - 最后消息预览

2. **对话详情**
   - 消息时间线
   - 消息气泡（发送/接收）
   - 消息类型（文本/文件）

3. **发送消息**
   - 文本消息输入
   - 文件附件上传
   - 快捷回复

4. **权限控制**
   - 导师只能与自己的学生通信
   - 学生只能与自己的导师通信

#### 数据流向
```
消息列表
    ↓
GET /messages/
    ↓
返回消息列表（支持分页和方向过滤）
    ↓
前端聚合为对话列表

对话详情
    ↓
GET /messages/conversation/:user_id
    ↓
返回与特定用户的所有消息
    ↓
自动标记接收的消息为已读
    ↓
前端渲染对话界面

发送消息
    ↓
POST /messages/
    ↓
{ receiver_id, content, message_type, file_url }
    ↓
验证权限（can_contact_user）
    ↓
创建消息记录
    ↓
返回消息详情
```

#### 前后端交互
| 操作 | 前端方法 | 后端API | 请求体 | 响应数据 |
|------|----------|---------|--------|----------|
| 加载消息列表 | loadMessages() | GET /messages/ | - | { items: Message[], total, page, per_page } |
| 加载对话 | loadConversation(userId) | GET /messages/conversation/:user_id | - | { items: Message[], total, page, per_page } |
| 发送消息 | sendMessage() | POST /messages/ | { receiver_id, content, message_type, file_url } | Message |
| 标记已读 | markAsRead() | GET /messages/:id | - | Message (自动标记已读) |

    ↓
GET /notifications/
    ↓

返回成功消息

| 全部标记已读 | markAllAsRead() | PUT /notifications/mark-all-read | - | { message: string } |

     │                    │                     │
     │                    │                     ↓
     │                    │              ┌──────────────┐
     │                    │              │ProgressReport│
     │                    │              │              │
     │                    │              └──────────────┘
     │                    │                     │

## 六、关键技术问题与解决方案

### 6.1 前端API端口配置问题

**问题**: 前端API配置指向localhost:4000，但后端运行在localhost:5000

**解决方案**:
```typescript
// frontend/src/utils/api.ts
const API_BASE_URL = 'http://localhost:5000';  // 修改为5000端口
```

或使用环境变量：
```typescript
const API_BASE_URL = import.meta.env.VITE_API_BASE_URL || 'http://localhost:5000';
```

---

### 6.2 认证Token管理

**问题**: Token需要在所有API请求中传递

**解决方案**:
- 使用全局authToken变量
- 在AuthContext中管理Token
- 登录成功后调用setAuthToken()
- 所有请求自动添加Authorization头

---

### 6.3 权限控制

**问题**: 确保导师只能访问自己的学生和数据

**解决方案**:
- 后端API验证current_user.role
- 查询时过滤mentor_id
- 使用can_contact_user()函数验证消息权限
- 前端路由保护（ProtectedRoute）

---

### 6.4 并发请求处理

**问题**: 工作台需要并发请求多个API

**解决方案**:
```typescript
const [studentsData, progressData, tasksData, ...] = await Promise.all([
  myApi.getMyStudents(),
  myApi.getMyPendingProgress(),
  taskApi.getTasks(),
  ...
]);
```

---

### 6.5 数据一致性

**问题**: 前端数据更新后需要同步

**解决方案**:
- 操作成功后重新加载数据
- 使用乐观更新提升用户体验
- 添加加载状态和错误处理

1. **认证流程**
   - 登录成功返回Token
   - Token正确传递到后续请求
   - Token过期处理

2. **学生管理**
   - 导师只能看到自己的学生
   - 学生详情包含进度历史
   - 搜索过滤功能正常

3. **进度审阅**
   - 待审列表只显示pending状态
   - 反馈提交后状态变为reviewed
   - 评分和审核结果正确保存

4. **任务管理**
   - 导师创建任务并分配学生
   - 学生提交任务
   - 导师反馈任务

5. **消息权限**
   - 导师只能与自己的学生通信
   - 学生只能与自己的导师通信
   - 消息已读状态正确更新

### 7.2 性能测试

1. **并发请求**
   - 工作台并发加载多个API
   - 响应时间 < 2秒

2. **大数据量**
   - 学生数量 > 100
   - 进度记录 > 1000
   - 消息记录 > 10000

3. **数据库查询**
   - 使用索引优化查询
   - 避免N+1查询问题
